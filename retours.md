# Faire un retour sur le guide

Ce guide est conçu comme un support de travail évolutif. Les retours permettent d’identifier les points à préciser, les formulations à améliorer ou les cas pratiques à ajouter.

## Deux manières de faire un retour

### 1. Commentaire LinkedIn

Le plus simple est de réagir directement en commentaire du post LinkedIn de publication, notamment pour une remarque courte, une question ou un cas d’usage général.

### 2. Remarque structurée via GitHub

Pour une remarque plus structurée, vous pouvez ouvrir une demande sur GitHub. Le formulaire prérempli permet d’indiquer le document concerné, le commentaire et la suggestion éventuelle.

[Proposer une remarque sur GitHub](https://github.com/danielkorabelnikov/kit-clause-limitative-responsabilite/issues/new?title=Retour%20sur%20le%20guide%20clause%20limitative%20de%20responsabilit%C3%A9&body=Bonjour%2C%0A%0AVoici%20mon%20retour%20sur%20le%20guide%20%3A%0A%0A-%20Document%20ou%20point%20concern%C3%A9%20%3A%0A-%20Commentaire%20%3A%0A-%20Suggestion%20%C3%A9ventuelle%20%3A%0A%0AMerci.)

## Types de retours utiles

- un point qui mérite une clarification ;
- une clause ou variante qui pourrait être ajoutée ;
- un cas d’usage rencontré en pratique ;
- une difficulté fréquente de négociation ;
- une remarque sur la lisibilité du guide.

## Précaution

Merci de ne pas partager d’informations confidentielles, de données personnelles, d’extraits de contrats non publics ou d’éléments permettant d’identifier une situation client. Les retours doivent rester généraux ou anonymisés.
