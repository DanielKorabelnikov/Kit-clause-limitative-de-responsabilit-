# Faire un retour sur le guide

[← Retour à l’accueil]({{ site.baseurl }}/)

Vos retours sont utiles pour améliorer ce guide et l’adapter aux questions rencontrées en pratique.

## Quels retours sont utiles ?

Vous pouvez notamment signaler :

- un point à préciser ;
- une clause qui mériterait une variante ;
- une difficulté fréquente de négociation ;
- une interaction avec un autre sujet contractuel ;
- une source à ajouter ;
- une question pratique rencontrée dans un contrat IT, SaaS ou numérique.

## Comment proposer un retour ?

Le plus simple est d’utiliser la page GitHub des remarques :

[Proposer / voir les remarques sur GitHub](https://github.com/danielkorabelnikov/Kit-clause-limitative-de-responsabilite/issues)

Sur GitHub, cliquez ensuite sur **New issue** si vous souhaitez créer une nouvelle remarque.

## Précaution

Ne publiez pas d’information confidentielle, de nom de client, de contrat non public ou de donnée personnelle inutile. Les remarques GitHub sont publiques si le dépôt est public.
