# Fiche de négociation — Plafond de responsabilité

[← Retour à l’accueil]({{ site.baseurl }}/)

Cette fiche vise à préparer la négociation du plafond de responsabilité dans un contrat IT, SaaS ou numérique.

## 1. Les positions fréquentes

### Position fournisseur

Le fournisseur recherche généralement :

- un plafond aligné sur le prix payé ;
- l’exclusion des dommages indirects ;
- une limitation globale et unique ;
- une cohérence avec son assurance ;
- une exposition proportionnée à sa marge et à son modèle économique.

### Position client

Le client recherche généralement :

- une sanction utile en cas de manquement grave ;
- une protection renforcée sur les obligations sensibles ;
- une couverture des risques opérationnels réels ;
- un régime adapté aux données, à la sécurité, à la continuité et à la réversibilité ;
- une clause qui ne neutralise pas les obligations essentielles.

## 2. Les arguments pour négocier un plafond plus élevé

- Le service est critique ou structurant.
- Le client dépend fortement du service.
- Le prestataire maîtrise les causes principales de risque.
- Le contrat comporte des engagements forts de disponibilité, sécurité ou continuité.
- Les données traitées sont sensibles ou stratégiques.
- Une défaillance peut avoir un effet significatif sur l’activité.
- Le prix ne reflète pas nécessairement l’ampleur du risque supporté par le client.

## 3. Les arguments pour négocier un plafond plus bas

- Le service est standardisé.
- Le prix est limité.
- Le service n’est pas critique.
- Le client conserve une maîtrise opérationnelle importante.
- Le fournisseur ne maîtrise pas toutes les causes de risque.
- Les obligations sensibles sont limitées.
- L’assurance du fournisseur ne permet pas de couvrir un plafond très élevé.

## 4. Les compromis utiles

Au lieu de choisir entre un plafond très bas et une absence de plafond, il est souvent possible de prévoir :

- un plafond général ;
- un plafond renforcé pour certaines obligations ;
- des exceptions limitées au plafond ;
- un régime spécifique pour la confidentialité, la sécurité, les données ou la propriété intellectuelle ;
- une option d’assurance complémentaire ;
- un plafond par catégorie de prestation.

## 5. Formule de négociation possible

```text
Le plafond général peut être accepté sous réserve de prévoir un régime distinct pour les obligations dont la violation exposerait le client à un risque disproportionné au regard du seul prix payé, notamment en matière de confidentialité, sécurité, données, réversibilité et propriété intellectuelle.
```

## 6. Point de vigilance

Le plafond ne doit pas être négocié isolément. Il faut le relire avec :

- les SLA ;
- les pénalités ;
- les obligations de sécurité ;
- les obligations de sauvegarde ;
- la réversibilité ;
- les assurances ;
- les exclusions de préjudices ;
- les cas de suspension et de résiliation.
